Basic step-by-step instructions to install and run the RCON application with Docker.

Up-to-date as of 28 Apr 2020

Installation & Setup Guide

The application requires the use of Docker to run. The first part of the instructions will cover the installation of the packages for docker & docker-compose. Most distributions will have suitable packages available within their respective repositories

The second part of the instructions will cover running the application and basic management of the containers.

Official Docker documentation can be found at the links below.

  • Docker Engine https://docs.docker.com/install/
  • Docker Compose https://docs.docker.com/compose/install/

Prerequisites

There are a couple of things that you will need to make this install as easy as possible.

  1. A copy of the latest docker-compose.yml file from the dev team.
  2. .env file containing your game server information.
  3. A pw folder which will contain the password you will create to access the RCON tool
  4. .htpasswd file to protect your rcon from unauthorised access.

Using the link below download the .zip file containing items 1-3 of the above list and extract the contents to a directory of your choosing i.e. /home/user.name/

Download
Note: The rcon will not be installed within this folder but all logs and data will be stored here for easy access.
The following instructions need to be entered into a command line interface (Terminal)

► Create a password to proctect your rcon

Change directory to the pw folder contained within the contents of the .zip file you extracted.

cd path/to/hll_rcon/pw/

Enter the command below but replace USER with your desired user& broadcasts name and press Enter. You will be prompted to enter a password and then verify it. If you would like to create multiple users, repeat this step.

printf "USER:`openssl passwd -apr1`\n" >> .htpasswd

Return back to the main folder.

cd ..

► Install docker & docker-compose to your system

Debian/Ubuntu Linux:

sudo apt-get install docker docker-compose

Arch Linux:

sudo pacman -S docker docker-compose

► Set Docker to run at start-up

sudo systemctl enable docker

Note: The Docker daemon (dockerd) runs under the root user. To be able to manage your Docker containers all of the following commands will need to be run using sudo.
If you do not wish to elevate privileges when executing docker commands, you will need to follow the instructions provided by Docker which can be found here. The remainder of this guide will assume the need for the use of sudo.

► Edit .env file

There are two easy ways to edit these files. Either via the command line (nano/vim) or using a graphical text editor gedit/kate.
See below examples.

Using nano:

nano .env

Once the .env opens you will see the below entries in the file.

HLL_HOST=
HLL_PORT=
HLL_PASSWORD=
HLL_DB_PASSWORD=
HLL_DB_HOST_PORT=5432
HLL_REDIS_HOST_PORT=6379
RCONWEB_PORT=8010

You only need to enter details for the first 4 settings. The last 3 only need to be changed if you have issues or another service already running on the designated port. Simply add your server details after the = sign and save.

Note: Make sure there are no spaces before or after the details you enter.

  1. Press Ctl+O to write out (save) your changes followed by Enter to confirm
  2. Press Ctl+X to close the text editor

Using vim:

vim .env

Once vim opens you will need to press a to enter INSERT mode and append to the end of the line already created. Enter the required details then press Esc once done. To save them type :wq and press Enter.

The last way to edit the .env file is to a graphical text editor. Navigate to the hll_rcon folder and press Alt+. to show hidden files. Double click the file you want to edit and your system will open the file in the default text editor.

► Create the containers and run them

This is a simple command becasue all of the required information is within the docker-compose.yml file.

sudo docker-compose up -d

The installation itself may take some time depending on the hardware and internet connection. Please be patient, you will see the progress from within the terminal.

Once complete your rcon will be available at your.ip.address:8010

► Container management and updates

To update your rcon to the latest version you will need to perform 3 seperate commands. Firstly you will need to stop the running containers, download the updates and then start the containers again.

Note: You will need to navigate to your hll_rcon folder before running the below commands.

sudo docker-compose down

sudo docker-compose pull

sudo docker-compose up -d
